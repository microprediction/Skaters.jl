# Porting map: Python skaters -> Julia (Skaters.jl)

This package is a port of the Python reference (microprediction/skaters).
One rule governs everything: the parity gate. No module ships without
passing the pinned parity vectors at 1e-6.

## The parity gate

`parity/vectors.json` is generated by the Python reference and pinned to a
specific commit (recorded below). The gate replays every implemented
scenario through the Julia implementation and requires agreement to 1e-6:

```
julia --project=. test/parity.jl      # must print PARITY OK
```

Never commit red. The script runs from the repo root with the project
active. It parses the vectors with JSON.jl and compares the seven probes
(mean, std, logpdf@0.3, cdf@0.3, quantile@0.1, quantile@0.9, crps@0.3) for
every post-burn-in step and horizon of every scenario.

Current status: 52 scenarios, 96040 probe values, PARITY OK.

## Two traps carried over from the R and Rust ports

CPython 3.12+ `sum()` is Neumaier-compensated. The Python source uses
`sum()` for Dist normalisation, moments, log-sum-exp and several dot
products. Julia's built-in `sum` uses pairwise summation, which differs.
`src/mathx.jl` implements `fsum` (Neumaier) and it is used exactly where
Python spells `sum()`. Sequential `+=` accumulators in Python (pdf, cdf,
crps, the matrix helpers, the GARCH grid NLL) stay sequential here.

Python `dict` insertion order is semantic in `sticky`: the atom selection
is a stable sort by weight, so ties break by first-seen order. The value
table is kept as parallel insertion-ordered vectors (`vals`, `wts`) and
the top-atom selection uses a stable `MergeSort`.

## Numerics

Float math is IEEE f64 throughout. `erf` and `erfc` come straight from the
openlibm that Julia's Base math already bundles, reached by `ccall` (no
extra package). openlibm's `erf` matches CPython's own `m_erf` to about
1e-16, far inside the 1e-6 gate. The Acklam inverse-normal quantile is
ported verbatim from tails.py, including the single Halley polish step.

State is plain nested `Dict{String,Any}` (and plain vectors) so it stays
trivially serializable. `Dist` is an immutable struct of `(w, m, s)`
tuples; `SplicedDist` is a small mutable struct that memoizes its quantile
grid, and it is only ever an output, never part of returned state.

## Refresh ritual

This port is faithful to skaters as of the pinned commit. To refresh:

1. In the Python repo: `PYTHONPATH=src python parity/gen_vectors.py`.
2. Copy `parity/vectors.json` here as `parity/vectors.json`.
3. Run the gate. Port whatever new or changed module makes it green.
4. Update the provenance commit below.
